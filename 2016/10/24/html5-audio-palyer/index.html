HTML5 不能同时播放同一个资源文件的问题

目前可使用的技术

  • HTML5 Audio 标签
  • Web Audio API

HTML5 Audio 对象

  • Audio 对象是 HTML5 中的新对象。
  • Audio 对象表示 HTML audio 元素。

  • 主要方法

    方法 | 描述
    :—- | :—-
    addTextTrack()| 向音频添加新的文本轨道。
    canPlayType() |检查浏览器是否能够播放指定的音频类型。
    fastSeek() |在音频播放器中指定播放时间。
    getStartDate() | 返回新的 Date 对象，表示当前时间线偏移量。
    load() |重新加载音频元素。
    play()|开始播放音频。
    pause() |暂停当前播放的音频。

  • 更多关于 Audio 的资料

    Web Audio API

  • Web Auido API 是 JavaScript 中用于在网页应用中处理音频的一个高级应该用接口, 主要用于实现Web端的音频处理, 并可以同已存在的其他API相配合, 包括XMLHttpRequest, Canvas 2D 和 WebGL 3D API.

  • 简单使用例子

我的使用场景

  • h5 页面上若干乐器按钮
  • 点击发出乐器声音
  • 每个乐器声音大概 1s
  • 支持连续点击(连续出声,第二次点击声音跟上一次点击的声音不冲突)

遇见的问题

  • 使用 Audio 对象的 play 方法, 上一次没有播放完毕,是不可以进行第二次播放

解决办法

  • 弃用 Audio 标签/对象
  • 完全使用 Web Audio Api
  • 通过 Ajax 获取音频文件的 buffer
  • 每次播放使用新的 BufferSource

  • ios chrome / safari 页面载入后不播放 (需用户触发事件, 才会进行播放)
